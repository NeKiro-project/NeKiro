# Contract Design: A2A Profile Conformance

## Version Identity

- A2A wire protocol: `0.3.0`
- Go SDK: `github.com/a2aproject/a2a-go v0.3.15`
- NeKiro A2A Profile Schema: `0.2`

The protocol pin does not change. The profile schema changes because NeKiro now
publishes normative operation, state, and conformance requirements.

## Required Operations

| Wire method | Go client | Go server | Valid success kinds |
|---|---|---|---|
| `message/send` | `SendMessage` | `OnSendMessage` | Message or Task |
| `message/stream` | `SendStreamingMessage` | `OnSendMessageStream` | Message, Task, status update, artifact update |
| `tasks/get` | `GetTask` | `OnGetTask` | Task |
| `tasks/cancel` | `CancelTask` | `OnCancelTask` | Same Task in canceled state |

## Wire Conformance

Fixed fixtures MUST validate:

- JSON-RPC version, request/response ID match, and exactly one result or error;
- required request params and structurally valid concrete result kinds;
- SSE `Content-Type`, framing, event order, and terminal outcome;
- stable Task and Context identities across a stream;
- artifact append, `lastChunk`, stable Artifact ID, and arrival order;
- `tasks/get` history length and task-not-found behavior;
- `tasks/cancel` task-not-found and task-not-cancelable behavior;
- all five NeKiro context headers on outbound calls.

SDK decode success alone is not conformance. Zero-valued Tasks, arbitrary task
state strings, unknown/missing `kind`, mismatched identity, event after terminal,
and EOF without terminal are invalid profile behavior.

## Task State Policy

- Transient: `submitted`, `working`.
- Successful terminal: `completed`.
- Failure terminal: `failed`, `rejected`.
- Cancellation terminal: `canceled`.
- Recognized but unsupported: `auth-required`, `input-required`, `unknown`.
- Empty/unspecified is invalid.

Rejected maps to `AGENT_EXECUTION_FAILED`. Unsupported or invalid states map to
`A2A_PROTOCOL_ERROR`. Timeout is a platform Invocation outcome and not an A2A
Task state.

## Fixture Strategy

Fixtures live under `contracts/a2a-profile/v0.3.0/conformance/` with a manifest
recording operation, expected validity, expected wire result kind, expected Go
concrete type where applicable, and expected protocol error. Go tests run the
same fixtures through real `a2aclient` transport and `a2asrv` handler paths;
fixtures are not generated by those paths.
