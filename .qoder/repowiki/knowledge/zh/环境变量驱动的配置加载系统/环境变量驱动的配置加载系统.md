本仓库采用纯环境变量驱动的配置系统，所有运行时配置均通过 NEKIRO_* 前缀的环境变量注入，不使用任何配置文件（JSON/YAML/TOML）或第三方配置库。

核心架构
- 集中式加载入口：apps/control-plane/internal/config/config.go 提供 Load()、LoadDatabaseURL()、LoadInvocationRuntime() 三个顶层函数，分别负责 Control Plane 主配置、数据库连接与调用运行时参数。
- 严格校验优先于默认值：每个字段在读取后立即进行格式/范围/语义校验，失败即返回错误阻止启动，不存在缺省值回退策略。
- 无分层覆盖机制：不支持多环境文件合并、优先级覆盖或热重载；每次进程启动都从零解析环境变量。

配置项分组
- 服务监听：NEKIRO_LISTEN_ADDRESS，host:port 格式，端口 1–65535
- 数据库：NEKIRO_DATABASE_URL，postgres/postgresql 方案，强制要求用户名、密码、sslmode
- 外部认证：NEKIRO_AUTH_MODE + NEKIRO_DEV_AUTH_PRINCIPALS_JSON，仅支持 development-static，principals JSON 禁止重复键/ID/TokenSHA256
- 内部认证：NEKIRO_INTERNAL_AUTH_MODE + NEKIRO_INTERNAL_DEV_AUTH_PRINCIPALS_JSON，同上，用于 Router 与 Control Plane 通信
- 调用运行时：NEKIRO_ROUTER_INTERNAL_URL / NEKIRO_ROUTER_INTERNAL_BEARER_TOKEN / NEKIRO_GATEWAY_INVOCATION_REQUEST_MAX_BYTES / NEKIRO_GATEWAY_SSE_EVENT_MAX_BYTES / NEKIRO_GATEWAY_INVOCATION_DEADLINE_MS，URL 必须精确匹配 /internal/v3/invocations，token 仅允许可打印 ASCII 且不含空白

校验与安全性约定
- requiredEnv() 拒绝空值、前后空白字符
- requiredStrictInt64() 仅接受十进制数字字符串并限制上下界
- rejectDuplicateMembers() 自定义 JSON 扫描器，禁止对象中出现重复键
- decodePrincipals() 使用 DisallowUnknownFields() 拒绝未知字段，并要求 Token 为 64 位小写十六进制 SHA-256 摘要（明文 token 不得出现在配置中）
- validIdentifier() 限定 principal ID 仅允许 [A-Za-z0-9._:-]，长度 1–128

部署集成
- deploy/compose.yaml 通过 Docker Compose 将宿主机 .env 中的值映射到容器环境变量，并使用 ${VAR:?message} 语法在缺失时直接报错退出
- .env.example 列出所有必需变量并提供注释说明（如 URL 编码、只存 SHA-256 摘要等）
- main.go 的 serve 子命令在启动时依次调用 config.Load() 和 config.LoadInvocationRuntime()，任一失败即终止进程

开发者规则
1. 新增配置项：在 Config 或 InvocationRuntimeConfig 结构体中添加字段，并在对应的 Load* 函数中通过 requiredEnv/requiredStrictInt64 读取与校验，同时在 .env.example 与 compose.yaml 补充示例
2. 禁止引入配置文件：不要添加 YAML/JSON/TOML 配置加载逻辑，保持单一来源（环境变量）
3. 敏感信息不落地：Bearer Token 必须以 SHA-256 摘要形式存放，严禁明文写入任何持久化位置
4. 环境变量命名：统一使用 NEKIRO_ 前缀，避免与宿主或其他服务冲突
