本模块按主题划分目录而非代码分层：
- `decisions/` 采用 ADR（Architecture Decision Records）编号格式（0001–0006），每个文件独立记录一项技术或边界决策及其上下文、后果，是跨子系统的单一事实来源。
- `contracts/compatibility.md` 作为跨语言契约变更的兼容性约束入口，被 `handoffs/CURRENT.md` 引用为必须阅读项。
- `handoffs/CURRENT.md` 是当前工作区（Workspace Closure & Invocation Planning）的权威交接文档，串联 Spec 009/010/012、Issue #4/#5/#9 的交付状态、验证命令与恢复步骤，并显式禁止在未冻结 T001 前实现 Spec 010 运行时。
- `runbooks/local-development.md` 定义本地环境要求（Go 1.26+、Node.js 24、Corepack/pnpm 11.3.0、Docker Compose ≥2.20）、`.env` 必填变量表、Compose + `go run ./apps/control-plane/cmd/control-plane migrate up|serve` 启动流程以及 `-tags=integration` 集成测试数据库后缀强制规则。
- `architecture/` 与 `roadmap/` 分别承载阶段规格与后续计划，供 `CURRENT.md` 和 ADR 交叉引用。
依赖方向单向：各文档互相通过路径引用彼此，不反向依赖源码；`CURRENT.md` 是唯一聚合当前状态的入口，其他文档作为其引用的静态证据。
