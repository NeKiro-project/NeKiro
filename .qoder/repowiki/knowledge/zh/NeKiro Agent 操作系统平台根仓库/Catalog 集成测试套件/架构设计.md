本模块为 Go `integration` 构建标签下的单文件测试包 `catalog_test`，采用「进程外控制平面 + 真实数据库」的验收测试架构：
- 测试入口 `TestCatalogPostgreSQLAndHTTPAcceptance` 在独立 `_test` 数据库中重建 `catalog` schema，调用 `migrate up` 两次以覆盖 v1→v2 迁移路径，随后 `go build ./apps/control-plane/cmd/control-plane` 编译并 fork 出子进程作为被测服务。
- 通过 `NEKIRO_AUTH_MODE=development-static` 与 `NEKIRO_DEV_AUTH_PRINCIPALS_JSON` 注入三个静态主体（owner-a、owner-b、user）及一个内部 router token，配合 `NEKIRO_LISTEN_ADDRESS` 绑定随机空闲端口，避免对外部环境产生副作用。
- 所有 HTTP 交互经统一 `request()` 助手发起，响应体使用 `json.Decoder.UseNumber()` 解码至 `contracts.CatalogEntry` / `SearchAgentsResponse`，确保 JSON number 精度不被丢失；`assertPlatformError` 同时校验状态码、`PlatformErrorCode` 与 `x-nek-trace-id` 头。
- 数据断言直接走 `pgxpool` 对 `catalog.agent_versions`、`catalog.agent_identities`、`catalog.publication_clock` 等表执行 SQL，覆盖跨所有者替换冲突、禁用幂等、并发 publish/disable 竞态、延迟提交时钟序列化、游标遍历稳定性与 10,000 版本分页性能等场景。
- `fixtures/catalog/*.json` 提供两个最小 Agent Card（runtime-a echo、runtime-b translate），由 `readFixture` 加载并通过 `registrationEnvelope` 包装为 `{card: ...}` 注册载荷。
- 依赖方向单向：测试 → contracts 类型定义 → 外部 control-plane 二进制 → PostgreSQL，不引入任何被测业务包源码。
