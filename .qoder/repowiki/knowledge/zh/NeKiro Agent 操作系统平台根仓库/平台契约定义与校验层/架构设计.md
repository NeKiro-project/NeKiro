本模块作为所有子模块共享的契约边界：
- contracts.go 暴露全局常量（AgentCardSchemaVersion、NorthboundAPIVersion 等）与核心类型别名（如 `PlatformError = PlatformErrorV2`、`InvocationEvent = InvocationEventV02`），由上层各 API 层直接引用，确保版本一致性。
- validate.go 通过 `//go:embed` 将 schemas/*.json、openapi/*.yaml、a2a-profile/*、agent-card/v0.2/*、invocation/v1/*、invocation-runtime/v1/*、installation/v2/* 全部打包进二进制，并在 NewValidator() 中注册 schema ID 到资源映射，编译为 jsonschema.Schema 实例；每个领域对象都有对应的 Validate* / Decode* 方法，先做 JSON 严格解码（DisallowUnknownFields + requireJSONEOF），再走 schema 校验，最后执行语义规则（如 Agent Card 的权限/能力匹配、Installation 的状态机约束）。
- result_contracts.go 与 runtime_contracts.go 分别封装 invocation-result-stream-event 与 invocation-runtime 的多版本结果契约，被 Validator 组合复用，避免重复编译。
- 测试文件 *_conformance_test.go 与 *_integration_test.go 直接消费 embed.FS 中的 conformance 套件，对同一份 schema 进行端到端验证，形成“schema → Go 结构体 → 语义规则 → conformance 用例”的闭环。
