- 每个契约领域在 schemas/ 下维护独立 .schema.json 文件，并通过 https://schemas.nekiro.dev/<domain>/v<N> 形式的 ID 注册到 jsonschema 编译器。
- 对外暴露的类型使用 `type X = XvN` 别名指向当前活跃版本，旧版本保留在单独文件中供向后兼容读取。
- 所有可序列化的契约结构体都配套 Validate* 方法，内部顺序固定为：严格 JSON 解码 → schema 校验 → 语义规则校验，并返回带 InstancePath/Keyword 的结构化错误。
- conformance 测试以 manifest.json 组织，按 valid/invalid 分类，并由 *_conformance_test.go 统一加载运行，不依赖外部网络或文件系统。
