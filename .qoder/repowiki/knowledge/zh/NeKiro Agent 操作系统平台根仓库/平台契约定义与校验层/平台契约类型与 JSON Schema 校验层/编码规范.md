- 对外暴露的类型别名统一指向具体版本结构体（如 `type PlatformError = PlatformErrorV2`、`type InvocationEvent = InvocationEventV02`、`type A2AProfile = A2AProfileV02`），使上层只依赖稳定别名而内部可演进多版本。
- JSON 反序列化合一采用 `bytes.NewReader` + `json.NewDecoder` + `UseNumber()` + `DisallowUnknownFields()` + `requireJSONEOF()` 四件套，确保未知字段拒绝与尾随内容检测。
- 对 `json.Number` 字段的数值合法性（正整数、非科学计数法、无引号包裹）通过独立的 `validatePositiveJSONInteger` / `rejectQuoted*LimitValues` 手工解析校验，而非依赖 JSON Schema。
- 验证错误统一包装为带结构化信息的自定义 error 类型（`SchemaValidationError`、`SemanticValidationError`、`A2AConformanceAssertionErrorV02`），并提供 `Error()` 与可选 `Unwrap()`，便于上层区分错误类别。
- 嵌入资源统一通过 `//go:embed` 指令打包到 `embed.FS`，再通过 `fs.Sub` 切分子目录访问（如 `a2aConformanceCorpusRoot`），避免运行时文件系统路径泄露。
