根仓库采用多语言 monorepo 组织：Go 工程（`go.mod`）承载 `contracts/` 与未来 `apps/control-plane`、`apps/a2a-router`；pnpm workspace（`pnpm-workspace.yaml`）聚合 `apps/*`、`sdks/*`、`agents/*`、`tests/*` 下的 TypeScript 工件，由根 `vitest.config.ts` 统一扫描测试入口，`tsconfig.base.json` 提供跨包一致的严格 TS 基线。

跨子模块的编排通过三层契约驱动：
- `contracts/` 作为语言无关的 JSON Schema / OpenAPI / A2A Profile 事实来源，被 Go 与 TS 两侧共同消费，禁止任何子模块绕过它直接共享内部类型。
- `specs/` 按编号维护 Feature Spec，所有新增能力必须先落 spec 再写代码，形成 SDD 门禁。
- `tests/` 启动真实 Control Plane + PostgreSQL 对 Catalog v3 API 做端到端验证，是契约到实现的回归锚点。
- `docs/` 沉淀 ADR、架构决策与 runbook，为各子模块演进提供可追溯依据。

依赖方向由 `AGENTS.md` 强制约束：Console → Northbound API Contract → Control Plane → Internal API Contract → A2A Router → A2A Profile → Agent；Router 不得反向导入 Control Plane 内部模块，Agent SDK 仅依赖 contracts。根脚本 `pnpm check` 串联 `go test ./... && go vet ./... && pnpm typecheck && pnpm test && pnpm build`，作为统一的 CI 门禁。
